## 🐛 踩坑與修正（按時序）

### B1. ⚠️ start-all.bat 閃退根因
**現象**：用戶雙擊 start-all.bat 視窗瞬間關閉
**根因**：背景已有舊 dev server (PID 35728)，Next.js 16 偵測到 duplicate 會**立刻 exit 新實例**
**修法**：start-all.bat 加 Step 0 pre-flight cleanup，用 PowerShell `Get-NetTCPConnection` 掃 port 3000-3005 + WMI 抓 `node.exe + next` 孤兒進程

**未來提醒**：Next.js 16 vs 14 行為不同 — 14 是 fallback 到 3001 繼續跑，16 是 exit。寫 dev script 不能假設某一版行為。

---

### B2. ⚠️ stop-all.bat 只查 port 3000
**現象**：用戶說 stop-all.bat 「點了沒有用」
**根因**：原本 `netstat | findstr ":3000" | findstr "LISTENING"` 寫法
1. 只查 3000，但 Next.js 會 fallback 到 3001+
2. `findstr ":3000"` 會錯誤匹配到 `:30001` 之類的客戶端 port

**修法**：改用 PowerShell `Get-NetTCPConnection -LocalPort 3000,3001,3002,3003,3004,3005` + WMI 抓 `Name='node.exe' AND CommandLine contains 'next'`

---

### B3. ⚠️ Bash tool 跑 `taskkill /PID` 失敗
**現象**：`taskkill /PID 35728 /F` 報「無法辨識/PID」
**根因**：Bash tool 預設用 git bash，git bash 把 Windows 風格的 `/PID` 當成路徑 `/PID`（POSIX path），轉成 `C:/Program Files/Git/PID` 然後找不到
**修法**：改用 PowerShell tool 跑 `Stop-Process -Id 35728 -Force`

**未來提醒**：Windows 系統命令（taskkill / sc / reg / icacls）一律改用 PowerShell tool 跑，不要走 Bash tool。

---

### B4. ⚠️ `git add -A` 不小心 commit 進 photo/ 17 張 jpg
**現象**：commit `bc85994` 含 17 張 LINE 相簿照片，與用戶意圖不符（用戶只想 push 跑得起來的檔案）
**修法**：補 commit `f269c81` 用 `git rm --cached photo/*.jpg` 取消追蹤，並把 `photo/` 加進 .gitignore

**未來提醒**：對於 untracked dir 在 git add 時要審慎，可考慮用 `git add <具體檔案>` 而非 `-A`。或先 `git status --short` 確認再 add。

---

### B5. ⚠️ `cd app && npm install` 改變 Bash tool 的 working directory
**現象**：跑完 `cd app && npm install recharts` 後，後續 Bash tool 命令的相對路徑都錯誤
**根因**：Bash tool 文件明寫 "working directory persists between commands"。我用 `cd app` 之後沒有 `cd` 回來
**修法**：跑 `cd /c/RenStudio/case/washinmachine` 顯式回到根目錄

**未來提醒**：用絕對路徑（從專案根算）。如果一定要 `cd`，配對 `&&` 在同一個 Bash call 內，例如 `cd app && npm install && cd ..`，避免 cross-call 污染。

---

### B6. ⚠️ Supabase TypeScript generated types 把 many-to-one 標成 array
**現象**：build 報 `Type '{ name: any; }[]' is not assignable to type '{ name: string; }'`
**根因**：Supabase JS client 的 generated types 對所有 join 都當 array，不論 cardinality（many-to-one 應該是 single object）
**修法**：cast 透過 unknown：`as unknown as { ... } | null`

**未來提醒**：寫 Supabase select 帶 `customer:customers(...)` 之類的 join 時，type cast 一律走 `as unknown as T` 模式，省得 TS 抓不到。

---

### B7. ⚠️ recharts Tooltip/PieLabel 型別簽名
**現象**：`formatter={(v: number) => ...}` 報 `Type '(v: number) => ...' is not assignable to type 'Formatter<ValueType, NameType>'`
**根因**：recharts 自己的 callback 簽名 ValueType 是 `string | number | undefined`，寫死 number 會編譯錯
**修法**：不要寫死參數型別，讓 TS 推：`formatter={(v) => [\`${v}%\`, "回購率"]}`

**未來提醒**：第三方圖表庫 callback 簽名常常複雜，盡量讓 TS 推導；要顯式型別時看 d.ts 別憑印象。

---

### B8. ⚠️ `.env*` gitignore 萬用匹配誤殺 `.env.local.example`
**現象**：`.env.local.example` 在 git status 標為 ignored
**根因**：`app/.gitignore` 有 `.env*` 萬用匹配，吃掉了所有 `.env*` 開頭的檔
**修法**：加 `!.env.local` + `!.env.local.example` 反向 unignore

**未來提醒**：寫 gitignore 用 `.env*` 之類的萬用匹配時要特別注意 `.example` template 檔被誤殺。建議用具名比較安全。

---

### B9. ⚠️ Windows cmd.exe 對 timeout 在管道環境下噴錯
**現象**：`stop-all.bat ... | head` 噴 `ERROR: Input redirection is not supported`
**根因**：`timeout /t 2 /nobreak >nul` 需要實體 console；在 PowerShell tool 用管道時 stdin 被導向，timeout 拒絕執行
**修法**：實際雙擊用戶看不到這個錯。我用 PS 管道測試是 artifact，可忽略

**未來提醒**：測試 .bat 要直接執行不要走管道。若管道是必要，用 `ping -n 3 127.0.0.1 >nul` 取代 `timeout`。

---

### B10. ⚠️ Windows CRLF warnings 反覆出現
**現象**：每次 `git add` 都跳 `LF will be replaced by CRLF`
**根因**：Windows git core.autocrlf = true（預設），會自動轉換
**處理**：不處理 — 是 Windows 開發環境常態，不影響功能

**未來提醒**：不要為了消除這些 warning 動 .gitattributes，會引發更多問題。讓 git 自己處理。

1. **讀 AGENTS.md / CLAUDE.md** — Next.js 16 是 breaking changes、project 有特殊規則
2. **讀現有 migration 文件** — 對齊風格、註釋格式、RLS policy 寫法、enum 命名

4. **讀 memory MEMORY.md** — 用戶風格、過往決策

### 大工程的 workflow
1. **先 plan 再 code** — 用 AskUserQuestion 提早砸關鍵決策（schema 不可逆變動之前）
2. **分階段 checkpoint commit** — schema → admin UI → staff UI → dashboard，每個階段一個 commit，方便回退
3. **commit 前必跑 `npm run build`** — TypeScript 型別 / Next.js 邊界錯誤要先抓
4. **migration / RLS / Supabase types 是 3 個獨立風險點** — 都要單獨驗證

### 工具使用提示
1. **Windows 系統命令一律用 PowerShell tool**，不要走 Bash tool（會被 git bash 路徑誤判）
2. **絕對路徑優先**，要 `cd` 就把整個鏈條包在一次 Bash call 內
3. **大型重構不要 `git add -A`**，先 `git status --short` 看清楚再具名 add
4. **Supabase generated types 對 many-to-one 標成 array** → 一律 cast as `unknown as T`

### 自動化任務的儀式
1. 開頭 `TaskCreate` 把任務全攤平（避免漏做）
2. 每完成一個就 `TaskUpdate` 改 completed
3. 收工前用 `notify-rc.mjs` 寄信通知（不用 RC 起床看 Claude Code 視窗）
