# 淨新洗衣管理系統 — 演示流程

**對象**：老闆娘（與老闆）
**長度**：完整 15–18 分鐘 / 精簡 6–8 分鐘
**目的**：對應每個痛點 → 看到對應功能 → 認同這套系統可以取代現有 Excel 流程

---

## 演示前準備（演示前 3 分鐘）

| 步驟 | 操作 |
|---|---|
| 1 | 確認資料齊全：`cd app && npm run seed:demo:reset` 重灌一次 |
| 2 | 雙擊 `start-all.bat`，瀏覽器會自動打開 |
| 3 | 用 `ren.studio.dev@gmail.com` / `admin1234` 登入 |
| 4 | **另外開一個無痕視窗**（不要登入），等會兒演示師傅手機端用 |
| 5 | 將瀏覽器寬度調到 1280px 左右（後台模式看得清楚） |

> 演示時手邊放：手機（裝著相機/Google Maps，待會兒用師傅 PWA 看畫面），也可以直接用無痕視窗模擬。

# 完整版 15 分鐘

## ▶ 開場（30 秒）

> 「老闆娘您好，您現在的痛點主要有四個：客戶資料散在 Excel 難搜尋、月曆排案要手記、師傅回繳現金對帳花時間、有些客戶轉帳跟現金混在一起搞不清楚。我們做的這套系統就是把這四件事一次解決，而且**未來不用付任何月費**。我們今天用真實的 demo 資料走一輪您看看。」

## ① 顧客管理（2 分鐘）

**📍 路徑**：左側選單 → 顧客
**🎯 對應痛點**：① 搜尋只能用電話/編號 ② 一客戶多地址多機型整合難

| 操作 | 你說什麼 |
|---|---|
| 進 `/customers` 看到 15 個 demo 客戶 | 「您現在所有客戶都在這裡，**台中、彰化、南投、雲林**都有」 |
| 上方搜尋框輸入「**陳**」 | 「搜尋不只是電話，**姓名、電話、編號、備註全部可搜**」 |
| 縣市下拉選「彰化縣」、鄉鎮下拉選「員林市」 | 「您之前說想用『**彰化-員林區**』快速篩選，這就是」 |
| 清空篩選，點開 **陳先生** | 「來看一個複雜客戶」 |
| 在陳先生詳情頁指地址區 | 「他有**兩個地址**：員林家裡 + 彰化公司」 |
| 指機器區 | 「他有 3 樣東西要清：洗衣機、冷氣、沙發。**以前要查不同 Excel，現在一頁全部看到**」 |
| 點右上「編輯」→ 不改任何東西，回到列表 | 「您可以隨時改、加地址、加機器」 |

**🔑 重點話術**：「以前找客戶資料要翻 Excel，現在 3 秒鐘搞定。」

**🎯 對應痛點**：④ 一單可多機多服務、自動加總、自動編號

| 操作 | 你說什麼 |

| 指派師傅選「王大哥」 | |

## ④ ★ 師傅 PWA（3 分鐘）

**📍 路徑**：無痕視窗用 `borenchang+wang@gmail.com` / `admin1234` 登入
**🎯 對應痛點**：⑤ 師傅現場 key-in、現場知道是否收錢、現場看到備註

| 操作 | 你說什麼 |
|---|---|
| 切到無痕視窗，登入王大哥 | 「**師傅用自己手機開瀏覽器登入**，跟老闆娘是不同帳號」 |
| 看到 `/staff` 今日案件清單 | 「這就是師傅的手機畫面，**只看到自己被指派的案件**，別人的看不到」 |
| 指某個帶**紅色「未收款」**的案件 | 「**紅色 = 沒收錢**，師傅去現場一眼就知道要跟客戶收」 |
| 指某個**綠色「已收款-匯款」** | 「**綠色匯款 = 客戶已轉帳給您**，師傅不用收現金，省得搞混」 |
| 點開案件詳情 | 「點進去看詳情」 |
| 指電話那行 | 「**電話可以直接點一鍵撥**」 |
| 指地址那行 | 「**地址點下去開 Google Maps 導航**」 |
| 指備註 | 「您之前在客戶建檔寫的『住二樓需爬樓梯』，師傅在這看得到」 |
| 滾到最底，按「**我收到現金了**」 | 「收完現金按這顆，**狀態立刻變已收款-現金**」 |
| 看 badge 變成綠色「已收款-現金」+ 橘色「待回繳」 | 「同時跳出**待回繳**標籤，提醒師傅要交給老闆娘」 |
| 按「**完成此案件**」 | 「清洗完按完成」 |

**🔑 重點話術**：「師傅不用打電話跟您報告每個狀況，**畫面上都看得到**。」

## ⑤ ★★★ 師傅待回繳對帳（3 分鐘）— 殺手級功能

**📍 路徑**：切回 owner 視窗 → 師傅薪資 → 師傅待回繳
**🎯 對應痛點**：⑥⑦ 師傅回繳金額對不上、有些客戶轉帳混在一起

> 這是整個演示**最重要的 3 分鐘**，慢慢講。

| 操作 | 你說什麼 |
|---|---|
| 進 `/payroll/settlements` | 「這是您每天/每週**最頭痛的對帳時間**」 |
| 指右上「全部待回繳總額」 | 「**全公司今天還欠老闆娘 NT$ XX,XXX**，一眼看到」 |
| 指最大張的師傅卡片（例如王大哥 NT$ 11,800，4 筆） | 「王大哥來交錢時，**直接打開這張卡**」 |
| 念卡片裡每一筆 | 「20260511-001 王太太 1800、20260512-002 陳先生 4300、20260514-001 李董 4900…**總共 11,800**」 |
| 假裝師傅交來 11,800 | 「師傅交來 11,800，**金額一致**」 |
| 按「**全部標記已回繳**」 | 「按一下，這 4 筆全部標記已回繳，**從清單消失**」 |
| 卡片消失，總額減少 | 「不用每筆點，**一鍵完成**」 |

**情境演練**（演示金額不對怎麼辦）：
| 操作 | 你說什麼 |
|---|---|
| 指另一張卡片，假設林師傅 NT$ 5,800 | 「假設林師傅交給您 5,600，**少了 200**」 |
| 「我們就點開每一筆比對」 | （點其中一筆看明細） |
| 「比方說是不是有筆給了客戶折扣忘記改，或哪筆少算」 | 「您找到差異後改完，再標記已回繳」 |

**🔑 重點話術**：「您過去對帳要翻單據、回想哪一筆有特殊情況，現在**所有差異都在一頁上**。對帳時間從 30 分鐘變 1 分鐘。」

## ⑥ 三種收款情境 demo（1.5 分鐘）

| 情境 | 系統怎麼處理 |
|---|---|
| **A. 客戶現金付給師傅** | 師傅 PWA 按「我收到現金了」→ 自動進入「待回繳」清單 |
| **B. 客戶事先匯款給老闆娘** | 老闆娘建單時直接選「已收款-匯款」→ **自動標記免回繳**，師傅去到現場看到綠色匯款 badge 就知道不用收 |
| **C. 客戶說要現場匯款** | 師傅 PWA 按「客戶說已匯款」→ 自動切到匯款狀態 |

> 演示 A 跟 B 的差異最有感。可以打開兩筆訂單對照看。

# 精簡版 6–8 分鐘（時間不夠用這個）

按這個順序，每段控制 1 分鐘以內：

1. **顧客**：搜尋「陳」→ 點開陳先生 → 「**多地址多機型一頁看光**」
2. **月曆**：拖拉一個案件 → 「**一秒改期**」
3. **新增訂單**：選陳先生 → 自動帶地址機器 → 看總額即時算 → 「**自動化作業**」
4. **師傅 PWA**：切無痕視窗登王大哥 → 看紅色未收款 → 按「我收到現金了」→ 變綠
5. **★★★ 待回繳**：進 `/payroll/settlements` → 指師傅總額 → 按「全部標記已回繳」→ 「**對帳從 30 分鐘變 30 秒**」
6. **設定**：示範可自己改服務價格 + 加新帳號 → 「**您自己改不用找我**」

# 演示中要回答的常見問題（先準備）

| 客戶會問 | 回答 |
|---|---|
| 「我要付多少錢？」 | 「**主機免費**（用 Supabase + Netlify 免費額度，5 年用不完）。簡訊提醒功能未來如果要做，那一塊您要自己開一個簡訊公司帳戶，但那是之後的事。」 |
| 「資料安不安全？」 | 「資料庫存在 Supabase（背後是 AWS），**帳號是您自己開的，所有權歸您**。我們做完案子就交給您，未來想換廠商也帶得走。」 |
| 「師傅看得到別人的案子嗎？」 | 「**看不到**。資料庫層級就把師傅限制在自己被指派的案件，沒辦法繞過。」 |
