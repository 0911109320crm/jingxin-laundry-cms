重置 Demo 資料

  • 確認終端顯示「~300 筆訂單 created」
  • start-all.bat 啟動前台（或確認 dev server 運行中）
  • 用 owner 帳號 borenchang@gmail.com / admin1234 登入

瀏覽器設定

  • 主視窗：後台管理頁面（寬度 1280px）
  • 另開無痕視窗：準備登入師傅帳號（稍後用）
  • 縮放設為 100%，字型夠大

開場白
0:30
「老闆娘好，我們今天來看一套專門為洗衣機清潔工坊設計的管理系統。您現在的流程主要靠 Excel 和 Line， 我們這套系統把客戶管理、月曆排案、師傅回繳、即將到期提醒全部整合在一起，而且未來月費零元。 我們用真實的 Demo 資料，大約 300 筆訂單橫跨半年，走一輪您看看。」
1
儀表板 Dashboard
1:00
😣 痛點：每天早上不知道今天有幾件、哪幾個師傅要出門、本月業績跑到哪
操作說什麼
進入 /dashboard「一打開就看到今天最重要的資訊」
指向 5 個 KPI 數字「本月營業額、案件數、今日待完成件數、即將到期客戶、本月取消率」
指向「vs 上月 ↑↓」「跟上個月比，一眼看出業績趨勢」
指向師傅案量直條圖「每位師傅本月完成幾件、還有幾件待做，一目瞭然」
指向今日案件清單「今天 4 件案子，幾點、哪個客戶、哪個區域、哪個師傅，全部在這」
指向「師傅待回繳」按鈕「這個按鈕直接跳到現金回繳頁，這個稍後講」
💡 精簡版可跳過直條圖，只秀 KPI + 今日清單
2
全域搜尋 NEW
0:30
😣 痛點：想找某個客戶要先點進「顧客」頁再搜尋，不能從任何地方直接找
操作說什麼
在任意頁面按 ⌘K（Mac）/ Ctrl+K（Win）「這是全域搜尋，不管在哪個頁面都可以用」
輸入「陳先生」「輸入兩個字以上，馬上搜尋所有客戶」
點擊結果跳到客戶詳情「點一下直接跳過去，不用逐層點選單」
再按 ⌘K 輸入訂單編號「也可以搜尋訂單編號」
按 ESC 關閉「Escape 關掉，非常快」
💡 也可以點 Sidebar 左下角的「搜尋 ⌘K」按鈕觸發
3
顧客管理
1:30
😣 痛點：客戶資料散在多個 Excel，同一客戶有多台機器、多個地址要分別查
操作說什麼
進入 /customers「這裡有 25 位 demo 客戶，台中彰化南投雲林都有」
看 KPI：總客戶 / 本月活躍 / 即將到期「上方三個數字讓您掌握客群健康度」
搜尋框輸入「陳」「姓名、電話、編號、備註全部都可搜，不只是電話」
縣市選「彰化縣」→ 鄉鎮選「員林市」「區域篩選，您之前說想照地區分配師傅，這就是」
點「匯出 CSV」「可以匯出 Excel 直接開，UTF-8 BOM 格式」
清空篩選，點「陳先生」「來看一個有多台機器的客戶」
指「地址」區塊「員林家裡 + 彰化公司，兩個地址都存著」
指「機器」區塊「洗衣機、冷氣、沙發，三樣東西一頁全部看到」
指「服務歷史」清單「幾月幾號清了什麼、哪個師傅做的、金額多少，完整記錄」
指「累計消費」統計區「這位客戶總共消費多少、服務幾次、上次距今多久」
點「一鍵新增訂單」「這個按鈕自動帶入這個客戶，不用重新填」（然後返回）
4
月曆排案
2:00
😣 痛點：排案要手寫行事曆或 Excel，更改日期要整行刪掉重填
操作說什麼
進入 /calendar，月視圖「這是全月排案畫面，每個色塊代表一個師傅的案件」
指各師傅顏色說明「王大哥藍、林師傅綠、陳師傅橘、黃師傅紫，一眼分清楚」
切換師傅 tab：只看「林師傅」「可以只看單一師傅，今天有幾件一清二楚」
切回「全部」，hover 某個 event「hover 看完整資訊：訂單編號、客戶、電話、地址、服務、金額」
指右上角每日總額 badge「每一天的格子右上角顯示當天總營業額」
指左側「待派工」面板「左邊有 4 件還沒排日期的案件在等待」
從待派工拖一張到某一天「拖過去就排定了，自動指派這個師傅」
再把同一 event 拖到另一天「改日期也是直接拖，不用重新開表單」
切到週視圖「週視圖可以看到每件案子的時間區段」
點空白日格「點空白格直接開新訂單表單，日期自動帶入」（不送出，返回）
💡 這個功能通常是老闆娘最感興趣的，可以多花時間示範拖拉
5
訂單管理
1:30
😣 痛點：訂單用 LINE 記，收款方式要另外用筆記本對，很容易漏
操作說什麼
進入 /orders「訂單列表，約 300 筆，半年的資料都在這」
指 KPI：筆數 / 總金額 / 完成金額「這三個數字是目前篩選範圍的統計，會隨篩選條件變動」
搜尋框輸入「陳先生」「可以搜客戶名稱、訂單編號、備註」
狀態下拉選「取消」「篩出所有取消的單，還可以看取消原因」
清空篩選，點一筆已完成訂單「來看訂單詳情」
指訂單詳情各欄位「服務項目、指派師傅、收款方式、回繳狀態，一頁搞定」
點「複製為新訂單」「同一客戶下次再來，點這裡複製，服務內容都帶過去，日期清空」（不送出，返回）
回列表，點「新增訂單」「來示範新增一筆」
選客戶「陳先生」「選完客戶，地址和機器自動帶出」
加一個服務項目，看金額試算「選服務後價格自動帶預設值，總金額即時計算」
設定時長 chip（如 90 分鐘）「選時長，結束時間自動算出，不用自己算」
關掉（不送出）「這樣一筆訂單幾十秒就建完了」
6
收款 & 師傅回繳
1:00
😣 痛點：師傅收了現金，老闆娘不確定有沒有回來，逐筆對帳花很多時間
操作說什麼
進入 /payroll/settlements（師傅待回繳）「這裡按師傅分組，每個師傅現在手頭還有多少現金沒有回繳」
展開其中一位師傅卡片「展開可以看每一筆訂單的詳細金額」
點「全部標記已回繳」「師傅把錢交回來了，點這一個按鈕，所有筆一次核帳完成」
指匯款訂單的「免回繳」badge「客戶直接匯款的，系統自動設成免回繳，不會混進去」
「以前每個月光對師傅的現金就要花一個晚上，現在一個按鈕三秒鐘。」
7
師傅手機 PWA
1:00
😣 痛點：師傅要打電話問老闆娘今天幾點去哪、客戶電話是多少
操作說什麼
無痕視窗登入 borenchang+wang@gmail.com / admin1234「這是師傅的手機介面，跟後台完全分開」
指今日案件清單「今天幾點要去哪、哪個客戶，師傅只看得到自己的案件」
點開一件案子「客戶地址、電話、備註（住二樓/養狗）都在這」
指「撥打電話」＋「Google Maps 導航」按鈕「點電話直接撥，點地圖直接開導航」
指三個現場按鈕「收到現金點這個、客戶說已匯款點這個、做完點完成，三個按鈕解決現場所有操作」
指頂部待回繳金額 banner「師傅自己也看得到手頭還有多少現金要回繳，不會忘」
切到「我的薪資」tab「師傅可以自己查本月計件薪資，每天明細都有」
💡 如果有實體手機，可以直接 demo 手機畫面更有說服力
8
即將到期提醒
0:45
😣 痛點：每年要翻 Excel 找哪些客戶快一年沒清了，手動一個個發 LINE
操作說什麼
進入 /reminders「系統自動偵測 11-13 個月沒服務的客戶，列出來讓您通知」
指「逾期 N 天」/ 「本週到期」badge「逾期多久一眼就知道，不用自己算日期」
點「複製 LINE 文案」「點這個按鈕複製提醒訊息，直接貼到 LINE 發給客戶」
點「已通知」按鈕「發完 LINE 後點已通知，記錄是誰什麼時候通知的」
切到「已通知」tab「通知過的客戶移到這個 tab，不會重複通知」
點「重新掃描」「隨時可以重新掃描，即時更新名單」
「以前每次要找這些客戶都要翻整份 Excel，現在系統自動幫您整理好，發 LINE 也只要點一下複製。」
9
營業報表
1:00
😣 痛點：月底要統計業績、哪些服務賣最好，還是要靠 Excel 手算
操作說什麼
進入 /reports「這是營業報表，預設本月」
指 4 個 KPI：營業額 / 完成件 / 服務客戶 / 平均客單「這四個數字最重要，本月業績一眼看完」
切換到「本季」「可以切換時間範圍：今日、本週、本月、本季、今年」
切到「自訂日期」選去年 12 月到現在「半年的完整資料都在，也可以自訂任意區間」
指「服務項目 TOP」「看哪個服務最賺錢，滾筒洗衣機還是分離式冷氣？」
指「區域熱區」「哪個鄉鎮客戶最多，幫您規劃師傅路線」
指「付款方式分佈」「現金 vs 匯款的比例」
指「VIP 客戶 TOP 10」「誰是您最重要的客戶，消費金額排名」
點「匯出 CSV」「一鍵匯出，Excel 直接開」
10
師傅計件薪資
0:45
😣 痛點：每月底要手算每個師傅做幾件、加大算幾台、折扣扣多少，很容易算錯
操作說什麼
進入 /payroll「師傅薪資頁，四位師傅的月報入口」
點「王大哥」進月報「這就是之前您用 Excel 做的版型，我們把它搬進來了」
指每日明細表格「每天去了哪裡、做了什麼、應收多少，完全自動統計」
指加大 / 折扣欄「加大跟折扣也自動計入，不用再手動加減」
點「下載 Excel」「點這個直接下載 .xlsx，版型跟您現在用的一樣」
切到上個月「可以看歷史月份，資料永遠存著」

收尾總結
0:30
「我們今天走了一輪，從客戶管理、月曆排案、訂單收款、師傅回繳、即將到期提醒、到月底薪資報表，所有流程都整合在一起。而且因為是 Supabase + Next.js 架構，每個月不需要額外付費。

接下來有兩件事需要您配合：一是確認四位師傅的真實姓名和帳號，二是如果有舊的 Excel 客戶資料，我們可以幫您一次性匯入，大約 3-4 個工作天完成。」

待辦事項提醒（需客戶配合）

  • 提供 4 位師傅真實姓名 + 手機號碼
  • 提供舊客戶 Excel 原檔（匯入約 3.5 天）
  • 決定網域：Netlify 預設 or 自訂網域
  • Supabase Dashboard 啟用 pg_cron（每日自動掃到期）
  • 提供公司 LOGO 圖檔（替換 PWA icon）

精簡版 6 分鐘（電話簡報或對方時間有限）

按照這個順序，每個段落各約 1 分鐘：

順序內容重點
① 儀表板5 個 KPI + 今日案件清單 — 強調「每天開電腦第一眼就看到」
② 全域搜尋按 Cmd+K 搜尋客戶 — 強調「不用點選單，隨時搜」
③ 月曆拖拉排案從待派工拖一張到日曆 + 師傅顏色分區 — 這個最有視覺衝擊力
④ 師傅 PWA手機端：今日案件 + 一鍵撥打 + 三個現場按鈕
⑤ 即將到期自動找出快一年沒清的客戶 + 一鍵複製 LINE 文案
⑥ 師傅薪資月報版型 + 下載 Excel — 強調「現在 Excel 做的全部自動化」
