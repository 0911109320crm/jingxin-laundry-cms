# 老闆娘手機 PWA spec（`/m` 路由）

## 目標

老闆娘大部分時間在手機上工作（不太開電腦）。她需要的 3 件事都要 1-2 tap 完成：

1. **快速查詢訂單** — 客戶來電對帳、查歷史
2. **快速調整師傅排程** — 客人改時間、塞單，需要看師傅空檔
3. **快速確認師傅回繳金額** — 每天結帳、知道誰還沒交錢

---

## 路由設計

```
/m            ─ 進入後預設選單頁
/m/search     ─ 訂單搜尋
/m/schedule   ─ 師傅排程
/m/settle     ─ 回繳檢核
```

整個 `/m` 是 owner/manager 專用，sidebar 不出現，純底部 tab 切換。

---

## 進入流程

老闆娘用 `ting201314` 登入後：
- 偵測到是 owner role → 自動 redirect 到 `/m`（手機）或 `/dashboard`（desktop）
- 偵測規則：`navigator.userAgent` 含 `Mobi` / `Android` / `iPhone` 就導 `/m`
- 老闆娘想看 desktop 版可手動切 `/dashboard`

## 共用元件

- `<BottomTabBar>` — 底部 4 tab 排程/搜尋/回繳/更多
- `<DateSwiper>` — 日期橫向切換 [<][今日][>]
- `<TechnicianTabs>` — 師傅橫向切換
- `<OrderCardMobile>` — 訂單摘要卡（compact）
- `<TimeSlotPicker>` — 排程改時段的 modal

## 不做的功能（v1 mobile）

- 新建訂單（電腦做比較好填）
- 編輯客戶資料（電腦做）
- 看報表 / 統計（電腦做）
- 月曆拖曳（手機體驗差）

老闆娘要做這些事 → 點底部「更多」 → 提示「請在電腦開啟 /dashboard」

## 技術細節

| 項目 | 做法 |
|---|---|
| 路由群組 | `app/(mobile)/m/...` 新 route group，layout 不含 sidebar |
| 樣式 | Tailwind responsive 強制 mobile，min-width 320px |
| 點擊區域 | 所有按鈕至少 44x44px (Apple HIG) |
| 字體 | 至少 16px (避免 iOS Safari 自動 zoom) |
| 離線快取 | 已有 service worker，照樣 work |
| 重要互動 | 確認 modal 而非 toast，避免誤觸 |
